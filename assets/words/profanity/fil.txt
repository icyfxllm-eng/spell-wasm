# Filipino (Tagalog) profanity block-seed — EXACT whole-word match (NFC,
# lowercased), boundary-safe so innocent words that merely CONTAIN these
# substrings (puti, putahe, bobojet…) are unaffected. This is a defensive
# User-Generated-Content filter for "My Words" (Apple Guideline 1.2): it blocks
# a child from ADDING these; it never appears in any word list or UI.
#
# Deliberately EXCLUDED to avoid false-blocking legitimate/neutral words:
#   hayop (animal), itlog (egg), pepe (a common name), pusa (cat), dumi (dirt),
#   and identity terms (bakla/bading/tomboy) — often neutral, not slurs.
#
# NOTE (follow-up, not fixed here per the audit work order): the per-language
# layer is EXACT-match, so it does NOT leet-fold or de-space like the English
# layer. Sophisticated evasions (g4g0 with mixed digits, s p a c e d letters)
# can still slip. The high-value fix is applying src/profanity.rs `normalize`
# to the per-language union check too — an architecture change, out of scope
# here. A handful of the most common numeric-leet forms are seeded below.
